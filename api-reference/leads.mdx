---
title: "Leads API — scores, leaderboard, and priority feed"
description: "Access lead scores, trigger manual scoring, browse the top-20 leaderboard, and fetch VERA's prioritized action list for your hottest prospects."
---

The Leads API surfaces VERA's lead intelligence layer. Each contact in your connected GoHighLevel CRM receives a composite score (0–100) built from six dimensions. You can query scores, trigger scoring on demand, and fetch a prioritized feed of who to call next.

---

## Lead score object

<ResponseField name="id" type="string">UUID of the lead score record.</ResponseField>
<ResponseField name="workspace_id" type="string">Workspace this record belongs to.</ResponseField>
<ResponseField name="client_id" type="string">VERA client ID. `null` for workspace-level leads.</ResponseField>
<ResponseField name="contact_id" type="string">GHL contact ID.</ResponseField>
<ResponseField name="contact_name" type="string">Full name of the contact.</ResponseField>
<ResponseField name="contact_email" type="string">Contact email address.</ResponseField>
<ResponseField name="score" type="integer">Composite lead score, 0–100. Higher is hotter.</ResponseField>
<ResponseField name="score_breakdown" type="object">Math scorer breakdown across six dimensions (see below).</ResponseField>
<ResponseField name="auto_score" type="integer">Automatic math scorer score. Runs on schedule without an AI call.</ResponseField>
<ResponseField name="ai_score" type="integer">Claude AI scorer score. Set on manual trigger only.</ResponseField>
<ResponseField name="ai_breakdown" type="object">Claude's reasoning breakdown. `null` unless manual scoring has run.</ResponseField>
<ResponseField name="ai_scored_at" type="string">ISO 8601 timestamp of last AI scoring run.</ResponseField>
<ResponseField name="source" type="string">Lead source: `ghl`, `meta`, `google`, `manual`, or `referral`.</ResponseField>
<ResponseField name="lead_source_detail" type="string">Additional source context.</ResponseField>
<ResponseField name="last_activity_at" type="string">ISO 8601 timestamp of last recorded activity.</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 timestamp of record creation.</ResponseField>
<ResponseField name="updated_at" type="string">ISO 8601 timestamp of last update.</ResponseField>

### score_breakdown object

The math scorer populates `score_breakdown` with six weighted dimensions:

```json
{
  "fit": 20,
  "engagement": 18,
  "timing": 15,
  "intent": 22,
  "recency": 12,
  "source_quality": 10
}
```

Individual dimension values sum to the overall `score`.

## List lead scores

<br />

`GET /api/v1/leads/{workspace_id}/scores`

Returns lead scores for your workspace, sorted by score descending.

### Path parameters

<ParamField path="workspace_id" type="string" required>
  Your workspace ID.
</ParamField>

### Query parameters

<ParamField query="client_id" type="string">
  Filter scores to a specific VERA client.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum number of results to return. Defaults to `50`, maximum `200`.
</ParamField>

<Tabs>

    curl "https://api.verahq.ai/api/v1/leads/ws_abc123/scores?client_id=clt_9f2e1a&limit=10" \

## Get a single lead score

<br />

`GET /api/v1/leads/{workspace_id}/scores/{contact_id}`

Returns the full score record for a specific GHL contact.

### Path parameters

<ParamField path="workspace_id" type="string" required>
  Your workspace ID.
</ParamField>

<ParamField path="contact_id" type="string" required>
  The GHL contact ID.
</ParamField>

```bash
curl "https://api.verahq.ai/api/v1/leads/ws_abc123/scores/ghl_contact_abc" \
  -H "X-API-Key: YOUR_API_KEY"
```

## Manually score a contact

<br />

`POST /api/v1/leads/{workspace_id}/score`

Runs Claude AI scoring for a single contact and upserts the result. AI scoring runs in its own lane — it never overwrites the automatic math scorer's `auto_score`.

### Path parameters

<ParamField path="workspace_id" type="string" required>
  Your workspace ID.
</ParamField>

### Body parameters

<ParamField body="contact_id" type="string" required>
  The GHL contact ID to score.
</ParamField>

<ParamField body="contact_data" type="object" required>
  Contact data to pass to the scorer. Recognized fields: `name`, `email`, `phone`, `tags`, `source`, `created_at`.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://api.verahq.ai/api/v1/leads/ws_abc123/score \
      -H "X-API-Key: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "contact_id": "ghl_contact_abc",
        "contact_data": {
          "name": "Sarah Johnson",
          "email": "sarah@example.com",
          "phone": "+15551234567",
          "tags": ["hot-lead", "consultation-booked"],
          "source": "meta",
          "created_at": "2026-04-10T09:00:00+00:00"
        }
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "contact_id": "ghl_contact_abc",
      "score": 91,
      "breakdown": {
        "fit": 22,
        "engagement": 20,
        "timing": 18,
        "intent": 24,
        "recency": 7,
        "source_quality": 0
      },
      "recommended_action": "Call within 24 hours — booked consultation and multiple hot-lead signals."
    }
    ```
  </Tab>
</Tabs>

## Trigger bulk scoring

<br />

`POST /api/v1/leads/{workspace_id}/score-all`

Pulls up to 100 contacts from your connected GoHighLevel account and runs the lead scorer on each. Results are upserted into the `lead_scores` table. Caps at 50 contacts per run.

Requires GoHighLevel to be connected in your workspace.

### Path parameters

<ParamField path="workspace_id" type="string" required>
  Your workspace ID.
</ParamField>

```bash
curl -X POST https://api.verahq.ai/api/v1/leads/ws_abc123/score-all \
  -H "X-API-Key: YOUR_API_KEY"
```

```json
{
  "scored": 48,
  "total_contacts": 98
}
```

## Lead priority feed

<br />

`GET /api/v1/intelligence/{workspace_id}/lead-priority`

Returns the top leads ranked by score with deterministic recommended actions derived from `score_breakdown`. No Claude call — purely derived from existing scores.

### Path parameters

<ParamField path="workspace_id" type="string" required>
  Your workspace ID.
</ParamField>

### Query parameters

<ParamField query="limit" type="integer">
  Number of leads to return. Defaults to `10`, maximum `50`.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl "https://api.verahq.ai/api/v1/intelligence/ws_abc123/lead-priority?limit=5" \
      -H "X-API-Key: YOUR_API_KEY"
    ```
