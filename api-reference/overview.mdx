title: "VERA REST API — overview and base URL"
description: "Base URL, versioning, authentication model, request and response format, and standard HTTP status codes for every VERA API endpoint."
---

The VERA REST API gives you programmatic access to every capability inside your Agency OS: clients, campaigns, leads, intelligence signals, action rules, and reports. All endpoints speak JSON and are organized under a consistent, workspace-scoped URL structure.

## Base URL

```
https://api.verahq.ai
```

Every API endpoint is prefixed with `/api/v1/`. Public routes — landing pages, tracking pixels, and health checks — sit outside this prefix.

## Versioning

The current version is **v1**. The version is part of the URL path (`/api/v1/`), so breaking changes will be introduced under a new version prefix rather than in-place.

## Authentication

All `/api/v1/` endpoints require an `X-API-Key` request header. See the [Authentication](/api-reference/authentication) page for details on obtaining your key and handling errors.

Public endpoints that do not require authentication:
- `GET /health` — liveness probe
- `GET /p/{slug}` — published landing pages
- `GET /track/{id}` — conversion pixel
- Webhook callbacks from Meta, GHL, and Stripe

## Workspace scoping

Most endpoints include `{workspace_id}` as a path segment. Every authenticated request is also validated against the workspace that owns the API key — passing a `workspace_id` that does not match your key returns `401 Unauthorized`.

## Request and response format

- All request bodies must be `Content-Type: application/json`.
- All responses are JSON.
- List endpoints return a wrapper object:

```json
{
  "items": [...],
  "count": 12
}
```

The wrapper key name matches the resource — for example, clients use `clients`, campaigns use `campaigns`, and so on.

## HTTP status codes

| Code | Meaning |
|------|---------|
| `200 OK` | Request succeeded |
| `201 Created` | Resource created (some POST endpoints return `200` with the created object) |
| `400 Bad Request` | Invalid request body or parameters |
| `401 Unauthorized` | Missing or invalid `X-API-Key` |
| `402 Payment Required` | Plan limit reached — upgrade required |
| `404 Not Found` | Resource does not exist or does not belong to your workspace |
| `422 Unprocessable Entity` | Request validation failed |
| `500 Internal Server Error` | Unexpected server-side failure |
| `502 Bad Gateway` | Upstream platform (Meta, Google) returned an error |

## Error response shape

```json
{
  "detail": "Client not found."
}
```

Plan-limit errors include additional fields:

```json
{
  "detail": {
    "error": "tier_upgrade_required",
    "current_tier": "solo",
    "required_tier": "agency",
    "message": "Solo plan is limited to 1 client. Upgrade to Agency to manage multiple clients.",
    "upgrade_url": "/settings/billing"
  }
}
```
