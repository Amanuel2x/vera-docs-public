---
title: "Reports API — generate and deliver client reports"
description: "Trigger AI-written client reports, retrieve as HTML or PDF, and email directly to clients. Weekly and monthly reports also auto-generate on schedule."
---

The Reports API lets you trigger, retrieve, and deliver client reports. Reports are generated by VERA's reporting skills and stored with full HTML and structured JSON content. You can render them as PDF, view the raw HTML, or email them directly to clients with the PDF attached.

VERA also generates reports automatically on a schedule: weekly summaries every Friday at 9 AM UTC, and monthly summaries on the 1st of each month at 7 AM UTC.

---

## Report object

<ResponseField name="id" type="string">UUID of the report.</ResponseField>
<ResponseField name="workspace_id" type="string">Workspace this report belongs to.</ResponseField>
<ResponseField name="client_id" type="string">VERA client ID the report was generated for.</ResponseField>
<ResponseField name="report_type" type="string">Type of report: `weekly_performance`, `monthly_summary`, `seo_audit`, or `campaign_review`.</ResponseField>
<ResponseField name="title" type="string">Report headline generated by VERA.</ResponseField>
<ResponseField name="content_json" type="object">Structured report data as a JSON object.</ResponseField>
<ResponseField name="period_start" type="string">ISO 8601 start of the reporting period.</ResponseField>
<ResponseField name="period_end" type="string">ISO 8601 end of the reporting period.</ResponseField>
<ResponseField name="sent_to" type="array">List of email addresses the report was sent to. `null` if not yet sent.</ResponseField>
<ResponseField name="sent_at" type="string">ISO 8601 timestamp of when the report was last emailed.</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 creation timestamp.</ResponseField>

## List reports

<br />

`GET /api/v1/reports/{workspace_id}`

Returns all reports for your workspace, ordered by creation date descending.

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>

### Query parameters

<ParamField query="client_id" type="string">Filter to a specific client.</ParamField>
<ParamField query="report_type" type="string">Filter by type: `weekly_performance`, `monthly_summary`, `seo_audit`, or `campaign_review`.</ParamField>
<ParamField query="limit" type="integer">Maximum results. Defaults to `50`, maximum `200`.</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl "https://api.verahq.ai/api/v1/reports/ws_abc123?client_id=clt_9f2e1a&limit=5" \
      -H "X-API-Key: YOUR_API_KEY"
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "reports": [
        {
          "id": "rpt_001",
          "workspace_id": "ws_abc123",
          "client_id": "clt_9f2e1a",
          "report_type": "weekly_performance",
          "title": "Apex Dental — Week of Apr 14: CPL Down 18%, Leads Up",
          "content_json": { "summary": "...", "sections": [...] },
          "period_start": "2026-04-14T00:00:00+00:00",
          "period_end": "2026-04-18T23:59:59+00:00",
          "sent_to": ["owner@apexdental.com"],
          "sent_at": "2026-04-19T08:05:00+00:00",
          "created_at": "2026-04-19T08:00:00+00:00"
        }

    }
    ```
  </Tab>
</Tabs>

## Get a report

<br />

`GET /api/v1/reports/{workspace_id}/{report_id}`

Returns a single report including `content_html` in addition to the standard fields.

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>
<ParamField path="report_id" type="string" required>The report UUID.</ParamField>

```bash
curl https://api.verahq.ai/api/v1/reports/ws_abc123/rpt_001 \
  -H "X-API-Key: YOUR_API_KEY"
```

## Generate a report

<br />

`POST /api/v1/reports/{workspace_id}/generate`

Triggers report generation for a client. VERA selects the appropriate skill based on `report_type` and enriches the output with Vera One network benchmarks when available.

| `report_type` | Skill used |
|---------------|-----------|
| `weekly_performance` | `weekly-summary` |
| `monthly_summary` | `reporting.monthly_report` |
| `seo_audit` | `seo-audit` |
| `campaign_review` | `campaign-monitor` |

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>

### Body parameters

<ParamField body="client_id" type="string" required>
  UUID of the client to generate the report for.
</ParamField>

<ParamField body="report_type" type="string" required>
  One of `weekly_performance`, `monthly_summary`, `seo_audit`, or `campaign_review`.
</ParamField>

<ParamField body="period_start" type="string">
  ISO 8601 start of the reporting period. Defaults to 7 days ago.
</ParamField>

<ParamField body="period_end" type="string">
  ISO 8601 end of the reporting period. Defaults to now.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://api.verahq.ai/api/v1/reports/ws_abc123/generate \
      -H "X-API-Key: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "client_id": "clt_9f2e1a",
        "report_type": "weekly_performance",
        "period_start": "2026-04-14T00:00:00Z",
        "period_end": "2026-04-18T23:59:59Z"
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "report_id": "rpt_002",
      "title": "Apex Dental — Week of Apr 14: CPL Down 18%, Leads Up"
    }
    ```
  </Tab>
</Tabs>

## Get report HTML

<br />

`GET /api/v1/reports/{workspace_id}/{report_id}/html`

Returns the rendered HTML for a report as `text/html`. Use this to preview a report in a browser or iframe before sending it.

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>
<ParamField path="report_id" type="string" required>The report UUID.</ParamField>

```bash
curl https://api.verahq.ai/api/v1/reports/ws_abc123/rpt_001/html \
  -H "X-API-Key: YOUR_API_KEY" \
  -o report.html
```

---

## Get report PDF

<br />

`GET /api/v1/reports/{workspace_id}/{report_id}/pdf`

Returns the rendered PDF as `application/pdf`. The PDF is generated from the stored report HTML and includes the client name in the header.

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>
<ParamField path="report_id" type="string" required>The report UUID.</ParamField>

```bash
curl https://api.verahq.ai/api/v1/reports/ws_abc123/rpt_001/pdf \
  -H "X-API-Key: YOUR_API_KEY" \
  -o report.pdf
```

## Email a report

<br />

`POST /api/v1/reports/{workspace_id}/{report_id}/send`

Emails the report to one or more recipients with the rendered PDF attached. The report is marked with `sent_to` and `sent_at` after delivery. A tracking pixel is embedded in the email HTML — opens are logged to the intelligence feed.

### Path parameters

<ParamField path="workspace_id" type="string" required>Your workspace ID.</ParamField>
<ParamField path="report_id" type="string" required>The report UUID.</ParamField>

### Body parameters

<ParamField body="to" type="array" required>
  List of recipient email addresses.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST https://api.verahq.ai/api/v1/reports/ws_abc123/rpt_001/send \
      -H "X-API-Key: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "to": ["owner@apexdental.com", "manager@apexdental.com"]
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "sent_to": ["owner@apexdental.com", "manager@apexdental.com"],
      "count": 2
    }
    ```
  </Tab>
</Tabs>

## Automatic report schedule

VERA generates reports on a recurring schedule without any API calls required:

| Report type | Schedule |
|-------------|---------|
| `weekly_performance` | Every Friday at 9 AM UTC |
| `monthly_summary` | 1st of each month at 7 AM UTC |

Auto-generated reports appear in the list endpoint and can be retrieved, previewed, and sent just like manually generated ones.
