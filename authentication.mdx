---
title: "Authenticate with the VERA API"
description: "VERA uses API key authentication for all REST endpoints. Learn how to find your key, pass it in requests, and handle authentication errors."
---

Every request to the VERA REST API must include your workspace API key. The key identifies your workspace and scopes all responses to the data you own. There are no user-level tokens — the API key represents the workspace as a whole.

## Getting your API key

Your API key is generated automatically when your workspace is created. To find it:

1. Log in to [app.verahq.ai](https://app.verahq.ai)
2. Go to **Settings → Workspace settings**
3. Copy the value shown in the **API key** field

If you need to invalidate your current key — for example, after accidental exposure — go to the same page and click **Rotate API key**. This generates a new key immediately and revokes the old one. Update any systems using the old key before rotating.

<Warning>
  Your API key provides full access to your workspace data. Do not include it in client-side code, commit it to version control, or share it publicly. Store it as an environment variable or in a secrets manager.
</Warning>

## Passing your API key

Include your API key in the `X-API-Key` request header on every call to a protected endpoint:

```bash
curl https://api.verahq.ai/api/v1/clients/{workspace_id} \
  -H "X-API-Key: your_api_key_here"
```

Replace `your_api_key_here` with your actual key and `{workspace_id}` with your workspace ID (also visible in **Settings → Workspace settings**).

The API also accepts the key as an `api_key` query parameter for OAuth redirect flows where adding a header is not possible — but for all standard API calls, use the header.

## Endpoints that do not require authentication

A small set of paths are intentionally open and do not require an `X-API-Key` header:

| Path pattern | Purpose |
|---|---|
| `/auth/*` | Signup and login |
| `/webhooks/*` | Inbound webhook callbacks from Stripe, Meta, GHL, and Calendly (verified by signature) |
| `/api/v1/meta/callback` | Meta OAuth redirect handler |
| `/api/v1/connect/callback` | GoHighLevel OAuth redirect handler |
| `/api/v1/fathom/callback` | Fathom OAuth redirect handler |
| `/api/v1/calendly/callback` | Calendly OAuth redirect handler |
| `/api/v1/free/*` | Public rate-limited tools (no key required) |
| `/health` | Health check |

All other `/api/v1/*` routes require a valid `X-API-Key` header.

## Authentication errors

If a request is missing a key or the key is invalid, the API returns:

```json
{
  "detail": "Invalid API key."
}
```

with an HTTP `401 Unauthorized` status. A missing header returns the same status with `"detail": "Missing X-API-Key header."`.

Check that you are:
- Including the `X-API-Key` header (not `Authorization` or another name)
- Using the key for the correct workspace
- Not sending a key that has been rotated or deleted
