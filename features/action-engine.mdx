The Action Engine lets you define rules that fire automatically when VERA detects something worth acting on. A CPL spike, a hot lead, a payment — instead of checking the dashboard manually, you set up a rule once and VERA handles it.

## How rules work

Every rule has three parts:

- **Trigger** — the event that activates the rule (e.g., a lead scores above 80, a CPL spike is detected)
- **Filter** — conditions the event must meet before the rule fires (e.g., score must be 80 or higher)
- **Action** — what VERA does when the rule matches (e.g., create a task in GoHighLevel, send an alert)

Rules also have a **cooldown** (minimum time between fires for the same entity) and a **daily cap** (maximum fires per day) to prevent duplicate actions.

## Default rules

The fastest way to get started is to click **Seed Default Rules** in the Automations page. VERA creates 8 pre-built rules instantly:

| What it does | When it fires |
|---|---|
| Pauses a campaign (requires your confirmation) | Campaign gets a 3x cost kill signal |
| Sends you an in-app alert | CPL spikes above threshold |
| Creates a follow-up task in GoHighLevel | A lead scores 80 or higher |
| Sends you an in-app alert | A payment is received via Stripe |
| Sends you an in-app alert | A client's churn risk score rises above 70 |
| Logs a funnel event | A GHL contact moves to "Closed" |

You can enable, disable, or edit any of these after seeding.

## Confirmation for campaign changes

Rules that pause or adjust campaigns default to requiring your approval before they execute. When one fires, VERA creates a **pending action** in your alerts feed — you click **Execute** to approve it, or dismiss it to skip.

<Warning>
  If you turn off confirmation for a pause rule, VERA will pause campaigns automatically without any human review. Only do this if you fully trust the trigger conditions.
</Warning>

## Test a rule before enabling it

Use the **Dry Run** button on any rule to simulate what it would do without actually executing the action. This is the safest way to verify a rule before turning it on.

<Tip>
  Start with the 8 default rules and let them run for a week before creating custom ones. Check the Execution Log to see exactly what's firing and how often — use that data to tune filters and cooldowns.
</Tip>
