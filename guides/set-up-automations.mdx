---
title: "Set up automated actions"
description: "How to use the Action Engine to respond to signals automatically — no manual monitoring required."
---

The Action Engine lets you set up rules that respond to events automatically. A hot lead comes in, VERA creates a follow-up task. A campaign CPL spikes, VERA sends you an alert. You define the rules once, and they run in the background.

## Start with the default rules

The fastest way to get going is to let VERA install a starter set of rules for you.

<Steps>
  <Step title="Open Automations">
    Click **Automations** in the left sidebar.
  </Step>

  <Step title="Seed the default rules">
    Click **Seed Default Rules**. VERA creates 8 pre-built rules instantly.
  </Step>

  <Step title="Review and toggle">
    Look through the rule cards. All 8 are enabled by default — turn off any that don't apply to your workflow.

    | Rule | What it does |
    |---|---|
    | CPL Spike | Sends you an in-app alert when CPL rises sharply |
    | Hot Lead | Creates a GoHighLevel task when a lead scores 80+ |
    | Payment Received | Sends you an alert when a Stripe payment is logged |
    | Churn Risk | Alerts you when a client's churn score crosses 70 |
    | Campaign Kill | Flags for your approval before pausing a struggling campaign |
    | Close | Logs a funnel event when a GHL contact moves to Closed |
  </Step>
</Steps>

## Create a custom rule

<Steps>
  <Step title="Click New Rule">
    Click **New Rule** in the Automations page to open the rule editor.
  </Step>

  <Step title="Choose a trigger">
    Pick what VERA should listen for:

    - **Lead scored** — fires when a lead's score is updated
    - **Campaign alert** — fires when a campaign alert is created (CPL spike, budget issue, etc.)
    - **GHL stage change** — fires when a contact moves to a new pipeline stage
    - **Payment received** — fires when a Stripe payment is logged
    - **Churn risk** — fires when a client's churn risk score crosses a threshold
  </Step>

  <Step title="Set a filter (optional)">
    Add a condition to narrow when the rule fires. For example, only fire when a lead scores 75 or higher, or only when the alert type is a CPL spike. Leave the filter blank to match everything.
  </Step>

  <Step title="Choose an action">
    Pick what VERA should do when the rule fires:

    - **Create a GHL task** — adds a follow-up task in GoHighLevel
    - **Tag a contact** — adds tags to a GHL contact
    - **Send an alert** — creates an in-app notification in VERA
    - **Send a Telegram message** — notifies a Telegram channel or user
    - **Run an AI skill** — triggers an AI analysis automatically
    - **Pause a campaign** — pauses a Meta or Google campaign (requires your confirmation)
    - **Enroll in a sequence** — adds a contact to an outbound email sequence
  </Step>

  <Step title="Set cooldown and daily cap">
    **Cooldown** sets the minimum time between firings for the same entity — so you don't get the same alert about the same client five times in a row. Default is 24 hours.

    **Daily cap** limits how many times the rule can fire total in a 24-hour window. Default is 10.
  </Step>

  <Step title="Test it, then enable">
    Click **Dry Run** to simulate what the rule would do without actually executing anything. Once you're satisfied, toggle **Enabled** on.
  </Step>
</Steps>

## Campaign actions need your approval

Rules that pause or adjust campaigns require you to approve the action before it executes. When one fires, VERA creates a **pending action** in your alerts feed. Click **Execute** to approve or dismiss it to skip.

<Tip>
  Run the default rules for a week before creating custom ones. Check the **Execution Log** to see what's actually firing — it shows each rule, what triggered it, and what happened. Use that to tune your setup rather than guessing.
</Tip>
